# P02: Token Lab 콘텐츠 데이터화

## 목표

Token Lab의 오버뷰(3-layer + 카테고리)와 본문 테이블용 카테고리·토큰 목록을 JSON으로 분리하고, `TokenOverviewDiagram` 및 `TokenLab.tsx`의 테이블 섹션이 해당 데이터를 사용하도록 한다.

## 구현 상세

### 1. 데이터 파일

- **overview.json**: `layers[]` — `{ id, title, note, categories: { label, desc }[] }`. 현재 `TokenOverviewDiagram`의 LAYERS와 동일 구조.
- **categories.json** (또는 overview에 통합): Shell / Global / Alias / Sys용 카테고리별 토큰 목록.
  - 예: `{ shellCategories: { title, tokens[], fullRow? }[], globalColorTokens, globalSpacingTokens, ... }`
  - 테이블과 오버뷰가 같은 소스를 참조하면, 토큰 추가 시 한 곳만 수정하면 됨.

### 2. 컴포넌트 수정

- `TokenOverviewDiagram.tsx`: LAYERS 하드코딩 제거, `content/labs/token-lab/overview.json` (또는 categories에서 layers 파생) import.
- `TokenLab.tsx`: SHELL_CATEGORIES, GLOBAL_* TOKENS, ALIAS_GROUPS, SYS_GROUPS를 JSON에서 로드. 필요 시 `fullRow` 등 플래그도 JSON에 포함.

### 3. 단일 소스

- 가능하면 오버뷰의 “카테고리 목록”과 본문 “테이블 카테고리”가 **동일 JSON 키**를 바라보게 하여, 카테고리 추가 시 오버뷰·테이블 모두 반영되도록 설계.
