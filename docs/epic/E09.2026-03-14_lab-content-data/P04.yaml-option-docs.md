# P04: (선택) YAML 지원 또는 문서화

## 목표

콘텐츠를 YAML로도 관리할 수 있도록 하거나, JSON 중심 워크플로우를 문서화하여 팀이 콘텐츠 수정 시 참고할 수 있게 한다.

## 구현 상세

### 옵션 A: YAML 지원

- `js-yaml` (또는 유사) 설치, 빌드 타임에 `*.yaml`을 읽어 JSON으로 변환하거나 Vite 플러그인으로 `.yaml` import 허용.
- `content/labs/**/*.yaml` 작성 가이드 및 기존 JSON과 동일 스키마로 샘플 제공.

### 옵션 B: 문서화만

- `docs/epic/E09.../` 또는 `src/content/labs/README.md`에 다음을 정리:
  - 각 랩별 JSON 파일 역할·스키마 요약
  - 항목 추가·수정 시 수정할 파일과 필드
  - (선택) 오버뷰와 본문이 같은 소스를 쓰는지 여부
