# P01: 콘텐츠 디렉토리 구조 및 Style Lab 데이터 이전

## 목표

`src/content/labs/` 디렉토리를 만들고, Style Lab 오버뷰용 데이터를 JSON으로 분리한 뒤 `StyleOverviewDiagram`이 해당 데이터를 import 하여 렌더링하도록 한다.

## 구현 상세

### 1. 디렉토리 및 설정

- `src/content/labs/style-lab/overview.json` 생성.
- 필요 시 `tsconfig`/Vite에서 `resolve.alias`로 `@/content` 또는 `@content` 매핑 (선택).

### 2. Style Lab overview.json 스키마

- **styleProperties**: `{ key, label, description, values[], cssVar }[]`
- **styleVariants**: `{ name, description, characteristics[], preview: { shadow, border, background } }[]`
- 현재 `StyleOverviewDiagram.tsx` 내 상수와 1:1 대응하도록 이전.

### 3. 컴포넌트 수정

- `StyleOverviewDiagram.tsx`: 내부 상수 제거, `import overview from '../../content/labs/style-lab/overview.json'` (또는 alias) 후 `overview.styleProperties`, `overview.styleVariants` 사용.
- 타입: `import type { StyleOverviewContent } from '...'` 또는 JSON에 맞는 interface를 같은 디렉터리 또는 `content/labs/types.ts`에 정의.

### 4. 검증

- Style Lab 페이지에서 오버뷰가 기존과 동일하게 표시되는지 확인.
