# P06: 색·크기 리터럴 검사 — ESLint + Stylelint 이관

## 목표

- `scripts/check-runtime-literals.js`에만 있던 **정규식 기반 검사**를 역할에 맞게 **분리**한다: **CSS·CSS Modules → Stylelint**, **TS/TSX·`domain/themes` 트리 → 기존 스크립트**(문자열·아트 예외 로직 유지).
- **아트 페이지**(`src/app/pages/art/**`)·정책상 예외가 스크립트·Stylelint `ignoreFiles`에 반영된다.
- **`npm run lint`** 한 번으로 ESLint + Stylelint + 리터럴 검사가 실행된다.
- 스크립트는 **lint 파이프라인의 공식 단계**로 고정; 별도 `npm run check:literals`는 동일 로직 단독 실행용으로 유지.

## 구현 상세

### 접근

1. **Stylelint:** `*.css` 대상, `ignoreFiles`에 아트·dist·node_modules. 규칙 세트는 최소(확장 규칙 도입은 저장소 성숙 후).
2. **TS/TSX·테마:** ESLint만으로 문자열 내부 `rgb(` 제외·아트 제외를 동일하게 맞추기 어려워 **`check-runtime-literals.js`를 유지**하고 lint에 포함.
3. **`package.json`:** `"lint": "eslint . && stylelint \"src/**/*.css\" && npm run check:literals"`.
4. **문서:** `yamang-design-stack` Commands에 lint 파이프라인 설명.

### 변경 범위 (실제)

- `package.json`, `eslint.config.js` (`coverage` ignore), `stylelint.config.mjs`
- `scripts/check-runtime-literals.js` (동작 유지)
- `.cursor/skills/yamang-design-stack/SKILL.md`
