# P07: CC-05 Code Hygiene — 주석·미사용 코드

## 목표

- **CC-05-01·02:** 구현부 인라인 설명·why·단계 나열을 제거하고, 유지할 맥락은 **`docs/design`** 등 공식 문서로 옮기거나 파일 상단 블록·참조 한 줄로만 남긴다. (원문: vibe_boilerplate `CC-05-code-hygiene`.)
- **CC-05-03·04:** 미사용 코드·미사용 import가 남지 않도록 정리한다(주석 작업과 병행 가능).
- E26 P06에서 샘플로 지적된 경로(테마·labs·Icon·추천 규칙 등)가 **basis와 정합**한 상태로 남는다.

## 구현 상세

### 접근

1. **우선순위:** `src/app/providers/ThemeProvider.tsx`, `useGlobalSettings`, `domain/palettes/mapping/recommendations.ts`, `domain/themes/token-set.ts`, Icon 데이터/컴포넌트, labs 중 설명용 인라인 주석이 많은 파일부터.
2. **삭제 우선:** 코드·이름만으로 드러나는 주석은 제거한다.
3. **이전:** 도메인 규칙·정책·긴 why는 `docs/design/` 또는 기존 아키텍처 문서에 단락을 추가하고, 해당 소스 파일 상단에 **문서 경로 한 줄**만 남긴다(CC-05-01 DO).
4. **예외(유지):** Sonar `NOSONAR`, ESLint `eslint-disable` 한 줄, 스택이 요구하는 **export 직상 JSDoc**, 부득이한 TODO+이슈 링크(CC-05-02 Exception).
5. **아이콘 카탈로그:** `material.ts` / `nucleo.ts` 등 대형 맵의 **섹션 구분**은 데이터 가독성용이다. 인라인을 없앨 경우 **파일 상단**에 “섹션 주석은 카탈로그 유지용” 한 줄로 범위를 고지하거나, 섹션을 별도 export로 쪼개는 방식 중 하나로 정한다.

### 변경 범위 (예상)

- `src/app/**`, `src/domain/**`, `src/shared/**` 중 CC-05 위반 샘플 및 인접 파일.
- 필요 시 `docs/design/` 보조 문서 1페이지 또는 `ARCHITECTURE.md` 앵커 추가.
