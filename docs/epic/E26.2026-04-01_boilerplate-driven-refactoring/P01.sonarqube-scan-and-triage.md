# P01: SonarQube 연결 및 이슈 분류

## 목표
- YamangDesign이 로컬 SonarQube에 프로젝트로 등록되어 스캔이 실행 가능하다
- 초기 스캔 결과에서 전체 이슈가 추출되고, vibe_boilerplate basis 규칙(CC, CN, TC, SC 등)별로 분류되어 있다
- 이슈 수정 우선순위(Critical → Major → Minor)가 정리되어 P03의 입력으로 사용 가능하다

## 구현 상세

### 접근
1. 로컬 SonarQube에 YamangDesign 프로젝트 등록
2. `sonar-project.properties` 작성 (React/TypeScript 프로젝트 설정)
3. sonar-scanner 실행하여 초기 스캔
4. 이슈 추출 및 SQ-02(Sonar Rule ↔ Basis Mapping) 기준으로 분류
5. 우선순위별 이슈 목록 정리

### 변경 범위
- **생성:** `sonar-project.properties` (프로젝트 루트)
- **생성:** 이슈 분류 결과 (본 문서 내)
- **수정:** `vitest.config.ts` (coverage 설정 추가)
- **참고:** 기존 `sonar-issues.json` (SonarCloud 결과, 비교용)
