# P06: 코드 컨벤션 점검

## 목표
- SonarQube가 커버하지 못하는 CN-01/CC-01~05 규칙의 준수 여부가 점검되어 있다
- 네이밍 패턴(레이어별 메서드명, 타입 서픽스, boolean 접두사)이 CN-01에 부합한다
- Orchestration/Computation 분리, 에러 분류, 데이터 흐름 규칙이 CC-02~04에 부합한다

## 구현 상세

### 접근

1. **네이밍 컨벤션 점검 (CN-01, Sonar 미커버 전량)**
   - **CN-01-01** 도메인 용어 직접 사용: `src/domain/` 내 타입/함수명이 도메인 용어(palette, theme, token, semantic 등)를 영어 원문 그대로 쓰는지
   - **CN-01-02** 레이어별 메서드명: domain(calculate/apply/validate), app(동사+명사 유스케이스), shared(유틸리티 관용구)
   - **CN-01-03** 타입 서픽스: DTO에 Request/Response, domain 타입은 서픽스 없음
   - **CN-01-06** boolean 접두사: `is`/`has`/`can` 사용 여부 grep
   - **CN-01-07** 약어 사용: 프로젝트 용어집 또는 universal 약어 외 사용 여부

2. **로직 레벨 점검 (CC-01, Sonar 미커버분)**
   - **CC-01-01** 선형 흐름: 주요 함수 샘플링 — 위에서 아래로 단방향 흐름인지
   - **CC-01-04** 점진적 상태 변환: 같은 의미의 변수를 재할당하는지 (불필요한 새 변수 생성 여부)
   - **CC-01-08** 순수 함수: `src/domain/` 내 함수가 외부 의존성을 인자로 받는지, 같은 입력→같은 출력인지
   - **CC-01-10** const 우선: `let` 사용 중 `const`로 대체 가능한 케이스 grep

3. **메서드 레벨 점검 (CC-02, Sonar 미커버 대부분)**
   - **CC-02-01/02** Orchestration vs Computation: `src/app/` 내 hook/handler가 비즈니스 계산을 직접 수행하는지
   - **CC-02-04** Application method = one use-case: 하나의 hook/handler가 여러 유스케이스를 처리하는지
   - **CC-02-05** Side effect 위치: domain 내 side effect(localStorage, fetch 등) 존재 여부

4. **에러 처리 점검 (CC-03, Sonar 미커버분)**
   - **CC-03-01** 에러 분류: 비즈니스 에러와 시스템 에러가 구분되어 있는지
   - **CC-03-02** Domain은 에러 생성만: domain 내 try-catch 존재 여부
   - **CC-03-04** Infrastructure 경계 변환: `src/app/infra/` 에서 외부 에러를 도메인 에러로 변환하는지

5. **데이터 흐름 점검 (CC-04, Sonar 미커버 대부분)**
   - **CC-04-02** Domain이 외부 표현 참조 금지: domain 내 DTO/API 응답 타입 import 여부
   - **CC-04-05** 전달 데이터 범위 최소화: 필요 이상의 props/데이터 전달 패턴 샘플링

6. **코드 위생 점검 (CC-05, Sonar 미커버분)**
   - **CC-05-01/02** 인라인 주석: 파일 상단 외 설명 주석 존재 여부 grep

### 변경 범위
- **수정 없음** — 점검 결과를 본 문서에 기록

## 체크리스트
- [ ] CN-01 네이밍 위반 목록이 기록되어 있다 (레이어별 메서드명, 타입 서픽스, boolean 접두사)
- [ ] CC-01 로직 레벨 위반 목록이 기록되어 있다 (선형 흐름, 순수 함수, const 우선)
- [ ] CC-02 메서드 레벨 위반 목록이 기록되어 있다 (Orchestration/Computation, side effect 위치)
- [ ] CC-03 에러 처리 위반 목록이 기록되어 있다 (에러 분류, domain try-catch)
- [ ] CC-04 데이터 흐름 위반 목록이 기록되어 있다 (domain 외부 참조, 전달 범위)
- [ ] CC-05 코드 위생 위반 목록이 기록되어 있다 (인라인 주석)
