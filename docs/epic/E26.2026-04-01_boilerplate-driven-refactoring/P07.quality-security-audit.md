# P07: 품질/보안 점검

## 목표
- 테스트 전략이 TC-01 기준(레이어별 전략, 동작 검증, 격리 원칙)으로 점검되어 있다
- 접근성이 AC-01 기준(Sonar 미커버분: 시맨틱 마크업, 키보드 접근성 심화)으로 점검되어 있다
- 보안이 SC-01/03/05 기준(프론트엔드 해당 규칙)으로 점검되어 있다

## 구현 상세

### 접근

#### 테스트 전략 점검 (TC-01)

1. **TC-01-01 레이어별 테스트 전략**
   - `src/domain/` 테스트: 순수 함수 단위 테스트가 존재하는지, TDD 대상인지
   - `src/app/components/` 테스트: 컴포넌트 렌더링/상호작용 테스트 존재 여부
   - `src/shared/utils/` 테스트: 유틸리티 함수 테스트 커버리지
   - 테스트가 없는 레이어/모듈 식별

2. **TC-01-04 동작 검증**: 기존 테스트가 구현 세부사항이 아닌 동작을 검증하는지 샘플링

3. **TC-01-06 테스트 격리**: 테스트 간 공유 상태 여부 확인

4. **TC-01-07 Mock 사용 기준**: 동일 레이어 내부를 mock하는 안티패턴 여부

#### 접근성 점검 (AC-01, Sonar 미커버분)

5. **AC-01-03 키보드 접근성 (심화)**: 커스텀 인터랙티브 요소(ColorPicker, Select, GenericTabs 등)가 Tab/Enter/Space/Esc로 조작 가능한지

6. **AC-01-04 시맨틱 마크업**: 주요 페이지 레이아웃이 `nav`, `main`, `header`, `section` 등 시맨틱 태그를 사용하는지 (div 남용 여부)

#### 보안 점검 (SC-01/03/05, 프론트 해당 규칙)

7. **SC-01-01 입력 검증**: 폼 입력, 사용자 데이터 처리 시 경계 검증 여부
8. **SC-01-03 출력 인코딩 (XSS)**: `dangerouslySetInnerHTML` 사용 여부 전수 스캔, 사용 시 sanitize 여부
9. **SC-01-04 허용 목록 기반 검증**: 검증 로직이 allowlist 패턴인지

10. **SC-03-01 민감 데이터 분류**: localStorage/sessionStorage에 민감 데이터 저장 여부
11. **SC-03-02 시크릿 미포함**: 소스 코드 내 API 키, 토큰 하드코딩 여부 grep
12. **SC-03-03 로그 민감 데이터**: `console.log`에 민감 데이터 노출 여부

13. **SC-05-01 CVE 스캔**: `npm audit` 실행 결과
14. **SC-05-02 Lock file**: `package-lock.json` 또는 `pnpm-lock.yaml` 커밋 여부
15. **SC-05-04 의존성 최소화**: 미사용 의존성 여부 (`depcheck` 또는 수동 확인)

### 변경 범위
- **수정 없음** — 점검 결과를 본 문서에 기록
- `npm audit` 결과는 실행 시점 스냅샷으로 기록

## 점검 대상 규칙

| 카테고리 | 규칙 | 점검 방식 |
|---------|------|----------|
| TC-01-01 | 레이어별 테스트 전략 | 파일 구조 + 커버리지 분석 |
| TC-01-04 | 동작 검증 | 테스트 코드 샘플링 리뷰 |
| TC-01-06 | 테스트 격리 | 테스트 코드 패턴 분석 |
| TC-01-07 | Mock 사용 기준 | grep — 동일 레이어 mock |
| AC-01-03 | 키보드 접근성 | 코드 리뷰 — 커스텀 컴포넌트 |
| AC-01-04 | 시맨틱 마크업 | grep — div vs semantic 비율 |
| SC-01-01/03/04 | 입력 검증, XSS, allowlist | grep + 코드 리뷰 |
| SC-03-01/02/03 | 민감 데이터, 시크릿, 로그 | grep 전수 스캔 |
| SC-05-01/02/04 | CVE, lock file, 미사용 dep | npm audit + 파일 확인 |
