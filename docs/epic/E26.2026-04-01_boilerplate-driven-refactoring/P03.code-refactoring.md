# P03: 코드 리팩토링

## 목표
- P01에서 식별된 SonarQube 이슈(Critical/Major/Minor)가 모두 해소된다
- 코드가 vibe_boilerplate basis 규칙(CC, CN, AR)에 부합한다
- SonarQube 재스캔 결과 clean 상태이다
- 기존 테스트가 모두 통과한다

## 구현 상세

### 접근
1. P01 이슈 목록을 우선순위 순으로 수정 (Critical → Major → Minor)
2. basis 규칙별 리팩토링:
   - **CC (코딩 규칙):** 순수 함수 원칙, 메서드 설계(단일 책임, 오케스트레이션 분리), 에러 핸들링 위치, 데이터 플로우(DTO/Entity 변환), 코드 위생(dead code 제거)
   - **CN (네이밍):** 레이어별 메서드 네이밍(domain: calculate/apply/validate, app: 유스케이스 동사, shared: 유틸리티), 타입 서픽스, boolean 접두사(is/has/can), 약어 규칙
   - **AR (아키텍처):** domain/app/shared 의존 방향 검증(app→domain→shared만 허용), 위반 import 수정
3. 수정 후 SonarQube 재스캔 → 잔여 이슈 확인 → 반복
4. 전체 테스트 실행 및 통과 확인
5. 필요시 리팩토링 과정에서 발견된 테스트 갭 보강

### 변경 범위
- **수정:** `src/domain/`, `src/app/`, `src/shared/` 전반 (SonarQube 이슈 + basis 규칙 위반 대상)
- **수정/생성:** 테스트 파일 (기존 테스트 수정 또는 누락 테스트 보강)
- SonarQube 이슈 수와 basis 위반 범위에 따라 변경 파일 수 결정 (P01 완료 후 구체화)
