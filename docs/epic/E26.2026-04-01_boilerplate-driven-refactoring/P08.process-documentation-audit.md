# P08: 프로세스/문서 점검

## 목표
- 깃 컨벤션이 GC-01 기준(브랜치 전략, Conventional Commits, 커밋 단위)으로 점검되어 있다
- 문서 체계가 DC-01~05 기준(문서 포맷, 계층, 에픽 작성 기준)으로 점검되어 있다
- 구현 패턴(PT-02), 워크플로(WF-01~03, RF-01), 에이전트 스킬(AG-01) 적용 현황이 확인되어 있다

## 구현 상세

### 접근

#### 깃 컨벤션 점검 (GC-01)

1. **GC-01-02 브랜치 네이밍**: 최근 브랜치 목록이 `{type}/E##/P##-{scope}` 또는 `epic/E##-{scope}` 패턴을 따르는지
2. **GC-01-03 Conventional Commits**: 최근 30개 커밋 메시지가 `type(scope): description` 형식인지 분석
3. **GC-01-04 커밋 본문**: 변경 이유와 명세 참조가 본문에 포함되어 있는지 샘플링
4. **GC-01-05 커밋 단위**: 한 커밋에 하나의 논리적 변경만 포함되어 있는지 샘플링

#### 문서 체계 점검 (DC-01~05)

5. **DC-01 문서 포맷**: 기존 에픽/Phase 문서(E16~E26)가 DC-01 고정 섹션 순서를 따르는지 샘플링
6. **DC-02 문서 계층**: `docs/` 구조가 3-tier 계층(basis/stack/project) 원칙에 부합하는지
   - YamangDesign은 basis를 vibe_boilerplate에서 참조, stack은 yamang-design-stack, project는 docs/design/
7. **DC-03 프로젝트 스케일 티어**: 프로젝트 티어(S/M/L)가 명시되어 있는지
8. **DC-04 프로젝트 문서 템플릿**: 도메인 용어집이 존재하는지 (palette, token, theme, semantic 등)
9. **DC-05 에픽/Phase 작성 기준**: 최근 에픽(E25, E26)이 DC-05 품질 기준(목표=완료 조건, 배경=As-Is/문제 분리, 체크리스트=결과 기반)을 따르는지

#### 구현 패턴 점검 (PT-02)

10. **PT-02-01 Strategy 패턴**: `src/domain/palettes/strategies/` 등 기존 패턴 사용이 PT-02 기준에 부합하는지
11. 기타 패턴(Factory, Builder 등)의 적용 여부와 적절성 확인

#### 워크플로 점검 (WF-01~03, RF-01)

12. **WF-01 Skill Gate**: 스킬 게이트가 AGENTS.md 또는 .cursor/skills/에 설정되어 작동하는지
13. **WF-02 Epic Lifecycle**: 에픽 번호(E##), Phase 번호(P##), 디렉토리 구조가 WF-02 규칙을 따르는지
14. **WF-03 Backlog**: `docs/epic/backlog/` 존재 여부, 백로그 관리 절차 확인
15. **RF-01 리팩터링 워크플로**: P03 리팩터링이 RF-01 절차(목표 고정, 커밋 단위, 보호 장치)를 따랐는지

#### 에이전트 스킬 점검 (AG-01)

16. **AG-01 Skill 작성 표준**: `.cursor/skills/` 내 skill 파일이 AG-01 기준(frontmatter, body 구조, 자기 완결성)을 따르는지 샘플링

### 변경 범위
- **수정 없음** — 점검 결과를 본 문서에 기록
