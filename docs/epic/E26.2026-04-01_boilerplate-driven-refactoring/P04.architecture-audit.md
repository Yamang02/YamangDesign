# P04: 아키텍처 점검

## 목표
- YamangDesign의 import 그래프가 AR-01-02(의존 방향) 규칙을 준수하며, 역방향 import가 0건이다
- 현재 3-layer 구조(domain/app/shared)의 선정 근거가 AR-01-01 기준으로 문서화되어 있다
- 컴포넌트 분류 체계가 AR-02-01 기준으로 명시되어 있고, 실제 배치와 일치한다
- 상태 관리(Context/hook)의 레이어 배치가 AR-02-02/AR-02-03에 부합한다

## 구현 상세

### 접근

1. **의존 방향 전수 스캔 (AR-01-02, AR-02-02)**
   - `src/domain/` 내 모든 파일에서 `@app/` import 여부를 grep — 역방향 위반
   - `src/shared/` 내 모든 파일에서 `@domain/` 또는 `@app/` import 여부를 grep — 역방향 위반
   - 허용 방향: `app → domain → shared` (단방향만)

2. **모듈 경계 점검 (AR-01-02)**
   - 레이어별 barrel export(`index.ts`) 존재 여부 확인
   - barrel을 우회하여 내부 파일을 직접 import하는 패턴 식별

3. **아키텍처 선정 근거 확인 (AR-01-01, AR-02-01)**
   - `docs/design/ARCHITECTURE.md` 또는 동등 문서에 3-layer 선정 이유가 기록되어 있는지 확인
   - AR-02-01이 요구하는 두 축(코드 구조, UI 컴포넌트) 선택이 명시되어 있는지 확인

4. **컴포넌트 분류 체계 점검 (AR-02-01)**
   - `src/app/components/`의 분류 기준(Atoms/Molecules/Organisms 또는 동등)이 문서화되어 있는지
   - 실제 컴포넌트 배치가 분류 기준과 일치하는지 샘플링 검증

5. **상태 관리 경계 점검 (AR-02-02, AR-02-03)**
   - `domain/themes/ThemeProvider.tsx`, `ThemeContext.ts`, `useTheme.ts` — React Context가 domain에 있는 것이 적절한지 판단
   - `app/context/`, `app/hooks/` — UI 상태와 비즈니스 상태의 분리가 AR-02-03에 부합하는지

### 변경 범위
- **수정 없음** — 점검 결과를 본 문서에 기록
- 위반 사항이 발견되면 수정 방향을 권장 사항으로 기록하고, 별도 에픽/백로그로 승격 여부를 판단

## 점검 대상 규칙

| 규칙 | 티어 | 점검 방식 |
|------|------|----------|
| AR-01-01 | Core | 문서 확인 — 선정 근거 존재 여부 |
| AR-01-02 | Core | grep 전수 스캔 — 역방향 import |
| AR-01-03 | Guide | 문서 확인 — 변경 이력 추적 여부 |
| AR-02-01 | Core | 문서 + 코드 — 분류 체계 명시 및 실제 일치 |

| AR-02-03 | Guide | 코드 리뷰 — UI/비즈니스 로직 분리 |

## 체크리스트
- [ ] `domain/` → `app/` 역방향 import가 0건이다
- [ ] `shared/` → `domain/` 또는 `app/` 역방향 import가 0건이다
- [ ] barrel export 우회 직접 import 목록이 식별되어 있다
- [ ] 아키텍처 선정 근거가 문서에 기록되어 있다 (또는 미비 사항이 기록됨)
- [ ] 컴포넌트 분류 체계가 문서화되어 있고 실제 배치와 일치한다 (또는 불일치 목록이 기록됨)
- [ ] Context/hook의 레이어 배치가 적절하다 (또는 부적절한 항목이 기록됨)
