# P01.5: CSS @layer Cascade 체계 도입

## 목표

기존 `/* @layer X */` 주석으로만 표현되던 CSS 우선순위를 실제 CSS `@layer`로 전환하여, shell / ds / preview / override 간 cascade 충돌을 브라우저가 강제하도록 한다. 이를 통해 `[data-shell] [data-context="preview"]` 복원 규칙과 ComparisonCard 로컬 테마 간의 우선순위 문제를 근본적으로 해결한다.

## 배경

P01에서 shell 토큰 격리를 완료했으나, ComparisonCard에서 glassmorphism surface 효과가 보이지 않는 문제가 남았다. 원인: `[data-shell] [data-context="preview"]` CSS 규칙이 surface 변수를 전역 값으로 복원하면서, 부모 요소의 inline `styleVars`를 자식 스코프에서 덮어쓴다.

이는 CSS custom property 상속의 특성상 **자식 요소에 직접 선언된 값이 부모에서 상속된 값보다 우선**하기 때문이다. `@layer`만으로는 상속 vs 직접 선언 문제를 해결할 수 없으므로, `@layer` + `data-scope="local"` 조합으로 해결한다.

## 설계

### 레이어 순서

```css
/* index.css 최상단 */
@layer reset, tokens, shell, ds, preview, override;
```

| Layer | 역할 | 대상 파일/규칙 |
|-------|------|---------------|
| `reset` | CSS 리셋 | `reset.css` |
| `tokens` | 변수 선언 (`:root`) | `variables.css`, `shell-variables.css`, `sys-variables.css`, `transitions.css`, `fonts.css` |
| `shell` | 앱 크롬 UI | `/* @layer shell */` 주석이 있는 31개 모듈 + `[data-shell]` 규칙 |
| `ds` | 테마 반응형 컴포넌트 | `/* @layer ds */` 주석이 있는 26개 모듈 |
| `preview` | preview 영역 surface 복원 | `[data-shell] [data-context="preview"]:not([data-scope="local"])` |
| `override` | 컴포넌트별 로컬 테마 | `/* @layer mixed */` 2개 모듈, 향후 로컬 스코프 규칙 |

### Cascade 충돌 해결

**변경 전:**
```
[data-shell]                     → surface 차단 (shell-variables.css)
  [data-context="preview"]       → 전역 surface 복원 (LabLayout.module.css)
    ComparisonCard (inline style) → 로컬 surface 값 (무시됨 — 상속 < 직접 선언)
```

**변경 후:**
```
@layer shell:  [data-shell]                                      → surface 차단
@layer preview: [data-shell] [data-context="preview"]:not([data-scope="local"]) → 전역 surface 복원
@layer override: (필요 시 로컬 스코프 규칙)

ComparisonCard: <div data-context="preview" data-scope="local" style={styleVars}>
  → :not([data-scope="local"])에 의해 전역 복원 규칙 미적용
  → inline styleVars가 그대로 유효
```

### CSS Modules + @layer 전략

각 `.module.css` 파일의 전체 내용을 `@layer` 블록으로 래핑:

```css
/* 변경 전 */
/* @layer shell — 크롬(--shell-*). */
.root { ... }

/* 변경 후 */
@layer shell {
  .root { ... }
}
```

Vite의 CSS Modules는 `@layer` 내부에서도 정상 동작한다 (class name hashing은 `@layer`와 독립적).

## 구현 상세

### 변경 파일

**글로벌 CSS (레이어 래핑):**
- `src/index.css` — `@layer` 순서 선언 + 기존 스타일 래핑
- `src/shared/styles/reset.css` — `@layer reset { ... }`
- `src/shared/styles/variables.css` — `@layer tokens { ... }`
- `src/shared/styles/shell-variables.css` — `:root` → `@layer tokens`, `[data-shell]` → `@layer shell`
- `src/shared/styles/sys-variables.css` — `@layer tokens { ... }`
- `src/shared/styles/transitions.css` — `@layer tokens { ... }`
- `src/shared/styles/fonts.css` — `@layer tokens { ... }`

**모듈 CSS (53개 — 주석 → 실제 @layer 전환):**
- `/* @layer shell */` 주석 → `@layer shell { ... }` (31개)
- `/* @layer ds */` 주석 → `@layer ds { ... }` (26개)
- `/* @layer mixed */` 주석 → `@layer override { ... }` (2개)

**Preview 복원 규칙 분리:**
- `src/app/layouts/LabLayout/LabLayout.module.css` — `[data-shell] [data-context="preview"]` 규칙을 `@layer preview`로 래핑 + `:not([data-scope="local"])` 조건 추가

**ComparisonCard 수정:**
- `src/app/layouts/LabLayout/ComparisonCard.tsx` — `data-scope="local"` + `style={styleVars}`를 `[data-context="preview"]` 요소로 이동
