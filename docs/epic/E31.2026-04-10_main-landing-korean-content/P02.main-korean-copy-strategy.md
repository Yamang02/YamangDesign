# E31 P02: Main Korean Copy Strategy

## 목표
- `Main`의 각 섹션이 한국어 사용자 관점에서 즉시 이해되는 정보 구조를 갖는다.
- 섹션별 역할(브랜드 소개/탐색 유도/신뢰 강화)이 카피 레벨에서 분명해진다.

## 구현 상세
- 분석 대상:
  - `src/app/pages/main/Main.tsx` 섹션 구조
  - `src/app/content/main-content.ts` 카피 데이터
- 변경 범위:
  - `src/app/content/main-content.ts` (문구 교체)
  - 필요 시 `src/app/pages/main/Main.tsx` (문구 길이에 맞는 마크업 미세 조정)

## 섹션별 분석 및 추천 문구

### 1) Hero
- 역할: 서비스 정체성 1문장 요약 + 즉시 행동 유도
- 현재 한계: 추상적 표현이 많아 "무엇을 할 수 있는지"가 늦게 드러남
- 추천 헤드라인(안 A, 권장):
  - 리드: `디자인 실험과 구현을`
  - 강조: `하나의 메인 허브`
  - 트레일: `에서 연결합니다`
- 추천 서브카피:
  - `Yamang Design은 팔레트·토큰·컴포넌트·아트 레퍼런스를 하나의 흐름으로 묶어, 아이디어를 즉시 검증 가능한 UI로 전환합니다.`
- CTA 추천:
  - 1차: `플레이그라운드 시작`
  - 2차: `레이아웃 둘러보기`

### 2) Modular Ecosystem (섹션 타이틀 + 설명)
- 역할: 메인에서 각 기능 영역으로 빠르게 분기시키는 내비게이션 허브
- 추천 타이틀: `모듈형 디자인 생태계`
- 추천 설명:
  - `각 섹션은 실사용 페이지로 바로 연결됩니다. 탐색이 곧 작업이 되도록 구조를 설계했습니다.`

### 3) Module Tiles (카드별)
- 역할: 영역 정의 + 클릭 의도 명확화

- Layouts
  - 제목: `레이아웃`
  - 설명: `랜딩·대시보드·아티클 구조를 기반으로 정보 위계를 빠르게 검증합니다.`
  - CTA: `구조 탐색`

- Labs
  - 제목: `랩`
  - 설명: `색상·타이포·간격·모션·반응형을 실험 단위로 점검합니다.`
  - CTA: `실험실 열기`

- Build
  - 제목: `빌드`
  - 설명: `아톰부터 오거니즘까지, 실제 조합 가능한 UI 인벤토리를 확인합니다.`
  - CTA: `프리미티브 보기`

- Context
  - 제목: `컨텍스트`
  - 설명: `서비스 맥락과 셸 규칙을 함께 살펴보며 화면 동작의 기준을 맞춥니다.`
  - CTA: `맥락 읽기`

- Art
  - 제목: `아트`
  - 설명: `레퍼런스 갤러리로 시각 언어를 확장하고 컴포넌트 감도를 높입니다.`
  - CTA: `갤러리 입장`

- Interactive Playground
  - 제목: `인터랙티브 플레이그라운드`
  - 설명: `토큰 조합과 UI 반응을 실시간으로 미리 보고, 결정 전에 빠르게 검증합니다.`
  - CTA: `샌드박스 진입`

### 4) Spotlight
- 역할: 메인의 신뢰 신호(왜 이 구조가 유효한지)
- 추천 타이틀: `아이디어에서 배포 가능한 UI까지`
- 추천 인용문:
  - `Yamang Design은 실험적 시각 언어를 토큰과 컴포넌트 체계로 연결해, 팀이 같은 기준으로 빠르게 구현하도록 돕습니다.`
- 메타 추천:
  - 저자: `Yamang Design Core`
  - 역할: `System Narrative`
- CTA 추천:
  - `디자인 설정 조정`
