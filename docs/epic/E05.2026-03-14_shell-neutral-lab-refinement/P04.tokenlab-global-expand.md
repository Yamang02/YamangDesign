# P04: TokenLab Global 섹션 확장

## 목표

TokenLab의 Global Tokens 섹션을 color 5개만 보여주는 상태에서
spacing, typography, motion 카테고리까지 포함한 전체 토큰 계층으로 확장한다.

## 구현 상세

### 현재 문제

`TokenLab.tsx` `GlobalSection`:

```tsx
const globalColorTokens = [
  '--ds-color-primary-500',
  '--ds-color-secondary-500',
  '--ds-color-accent-500',
  '--ds-color-neutral-500',
  '--ds-color-sub-500',
];
```

TokenLab의 목적은 "3-tier 토큰 계층 전체를 시각적으로 검사"하는 것인데,
Global Tokens가 color만 보여주면 spacing/typography/motion이 왜 Global인지
전달되지 않는다.

### 변경 방향

Global Tokens 섹션을 카테고리별로 분리. 색상이 아닌 토큰은
스와치 없이 토큰명·값만 표시하는 별도 `ScalarTokenCard` 사용.

**표시할 카테고리 및 대표 토큰:**

```
Color (현행 유지)
  --ds-color-primary-500, secondary-500, accent-500, neutral-500, sub-500

Spacing (대표 6개)
  --ds-spacing-1, -2, -3, -4, -5, -6

Typography (대표)
  --ds-text-xs, -sm, -md, -lg, --ds-font-sans, --ds-font-mono

Motion (대표)
  --ds-duration-fast, -normal, -slow, --ds-ease-easeOut, --ds-ease-productive
```

카테고리별 `<h3>` 구분 헤더 추가. 색상 카드는 기존 `ColorTokenCard` 유지,
나머지는 새 `ScalarTokenCard`:

```tsx
function ScalarTokenCard({ token }: { token: string }) {
  const value = useCssVar(token);
  return (
    <div className={styles.scalarCard}>
      <code className={styles.tokenName}>{token}</code>
      <span className={styles.tokenValue}>{value || '(not set)'}</span>
    </div>
  );
}
```

### Component Inspector 프리뷰 수정

현재 `ComparisonCard`가 `styleVars` 없이 렌더 → shell 고정 스타일.
테마 반응형으로 보이게 하려면 `data-context="preview"` 래핑 또는
현재 ThemeProvider 값을 반영한 `styleVars`를 주입해야 한다.

`useTheme()` 훅으로 현재 팔레트/스타일의 CSS 변수를 가져와
`ComparisonCard`의 `styleVars`로 전달. 이미 Playground에서 동일 패턴(`getThemeVariables`) 사용 중.

- `useTheme`: `themes/useTheme` (또는 `themes`)
- `getThemeVariables`: `constants` (또는 `constants/lab-presets`) — 시그니처 `(palette, style)`.

영향 범위: `TokenLab.tsx`, `TokenLab.module.css`
