# P05: Playground 패널 아키텍처 전환 + 컬럼 의도 명확화

## 목표

Playground의 두 컬럼을 독립 구성 상태를 가진 `PreviewPanel` 컴포넌트로 분리하여
확장 가능한 패널 아키텍처의 기반을 마련한다.
동시에 Brand / System 컬럼의 시각적 구분과 의도 전달을 개선한다.

## 구현 상세

### 핵심 타입 — `PlaygroundPanel`

```typescript
// src/pages/layouts/Playground/types.ts (신규)
export interface PlaygroundPanel {
  id: string
  label: string
  description: string
  palette: PaletteName
  style: StyleName
  font: FontKey
  systemPreset: SystemPresetName
  neutralPreset: NeutralPresetName
}
```

Playground 상태를 평면 값들 → `PlaygroundPanel[]`로 전환.
현재 2개 패널은 고정(Brand / System), 추가/삭제는 E07에서 구현.

### `PreviewPanel` 컴포넌트

```tsx
// src/pages/layouts/Playground/PreviewPanel.tsx (신규)
function PreviewPanel({ panel, variant }: { panel: PlaygroundPanel; variant: 'brand' | 'system' }) {
  return (
    <div className={styles.panel}>
      <PanelHeader panel={panel} />
      <div
        className={`${styles.previewContainer} ${variant === 'system' ? styles.previewContainerSystem : ''}`}
        style={{ ...getThemeVarsForPanel(panel), fontFamily: getFontFamily(panel.font) }}
      >
        <PanelContent variant={variant} />
      </div>
    </div>
  );
}
```

`PanelHeader`: 타이틀 + 부제를 `panel.label` / `panel.description`으로 구성.

### Neutral Scale 스와치 (Brand 패널)

Brand 패널 하단 섹션으로 추가:

```tsx
<section className={styles.previewSection}>
  <h3 className={styles.previewSectionTitle}>Neutral Scale</h3>
  <div className={styles.neutralSwatch}>
    {[100, 300, 500, 700, 900].map((step) => (
      <div
        key={step}
        className={styles.neutralSwatchItem}
        style={{ backgroundColor: `var(--ds-color-neutral-${step})` }}
        title={`neutral-${step}`}
      />
    ))}
  </div>
</section>
```
