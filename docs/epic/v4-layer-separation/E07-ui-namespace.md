# E07: UI 영역별 토큰 네임스페이스 분리

> **구현 완료** (2025-02)

## 목표

**쇼케이스 콘텐츠**와 **사이트 UI(컨트롤 영역)**를 별도 토큰 네임스페이스로 분리하여, 테마 전환이 각 영역에 독립적으로 적용되도록 한다.

---

## 배경

### 현재 문제

```
사용자가 "파스텔 테마"를 선택하면:
├── 쇼케이스 영역 → 파스텔 적용 ✓ (의도한 동작)
├── 헤더/컨트롤 UI → 파스텔 적용 ✗ (혼란 유발)
└── 접근성 설정 패널 → 파스텔 적용 ✗ (조작 어려움)
```

컨트롤 UI까지 테마가 바뀌면 사용자가 조작하기 어렵고, 쇼케이스와 실제 UI의 경계가 모호해진다.

### 해결 방향

```
--ds-*   → 쇼케이스용 (동적, 테마 전환 대상)
--ui-*   → 사이트 UI용 (고정, 헤더/설정패널/접근성 컨트롤)
```

---

## UI 영역 정의

### 1. 쇼케이스 영역 (`--ds-*`)

테마 시스템이 적용되는 동적 영역:

| 영역 | 설명 |
|------|------|
| **서비스 테마 프리뷰** | 사용자가 선택한 테마로 컴포넌트 데모 |
| **Admin 테마 프리뷰** | Admin 스타일 쇼케이스 |
| **기타 테마 프리뷰** | 실험적 테마, 다크모드 등 |

### 2. 사이트 UI 영역 (`--ui-*`)

테마와 무관하게 **항상 일관된 디자인**을 유지하는 영역:

| 영역 | 설명 | 현재 |
|------|------|------|
| **헤더** | 테마/팔레트 선택 드롭다운 | 있음 |
| **접근성 패널** | 고대비, 폰트 크기, 모션 감소 등 | 예정 |
| **설정 드로어** | 사이트 설정, 환경설정 | 예정 |
| **네비게이션** | 사이트 내 이동 | 있음 |

## 토큰 네임스페이스 설계

### `--ui-*` (사이트 UI 토큰)

```css
:root {
  /* 배경 */
  --ui-bg-base: #1a1a1a;
  --ui-bg-surface: #2d2d2d;
  --ui-bg-elevated: #3d3d3d;

  /* 텍스트 */
  --ui-text-primary: #ffffff;
  --ui-text-secondary: #a0a0a0;
  --ui-text-muted: #666666;

  /* 경계선 */
  --ui-border-default: #404040;
  --ui-border-subtle: #333333;

  /* 액션 (컨트롤 버튼 등) */
  --ui-action-primary: #6366f1;
  --ui-action-hover: #818cf8;

  /* 포커스 */
  --ui-focus-ring: #6366f1;
}
```

### `--ds-*` (쇼케이스 토큰) - 기존 유지

```css
[data-showcase] {
  /* 기존 디자인 시스템 토큰 사용 */
  --ds-color-bg-base: /* 테마에 따라 동적 */;
  --ds-color-text-primary: /* 테마에 따라 동적 */;
  /* ... */
}
```

## 접근성 확장 고려

### 현재 (MVP)

```
┌─────────────────────────────────────────┐
│  헤더 (--ui-*)                          │
│  [테마 선택 ▼] [팔레트 선택 ▼]           │
└─────────────────────────────────────────┘
```

### 미래 (접근성 기능 추가 시)

```
┌─────────────────────────────────────────┐
│  헤더 (--ui-*)                          │
│  [테마 ▼] [팔레트 ▼] [접근성 ⚙️]         │
└─────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────┐
│  접근성 패널 (--ui-*)                    │
│  ┌─────────────────────────────────┐   │
│  │ □ 고대비 모드                    │   │
│  │ □ 큰 글씨                        │   │
│  │ □ 모션 감소                      │   │
│  │ □ 색맹 모드: [선택 ▼]            │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

**동일한 `--ui-*` 네임스페이스**를 사용하므로 구조 변경 없이 확장 가능.

## 구현 전략

### Phase 1: 토큰 정의

```typescript
// src/tokens/ui/index.ts
export const uiTokens = {
  colors: {
    bg: {
      base: '#1a1a1a',
      surface: '#2d2d2d',
      elevated: '#3d3d3d',
    },
    text: {
      primary: '#ffffff',
      secondary: '#a0a0a0',
      muted: '#666666',
    },
    border: {
      default: '#404040',
      subtle: '#333333',
    },
    action: {
      primary: '#6366f1',
      hover: '#818cf8',
    },
    focus: {
      ring: '#6366f1',
    },
  },
} as const;
```

### Phase 2: CSS 변수 주입

```typescript
// src/styles/ui-variables.css
:root {
  --ui-bg-base: #1a1a1a;
  --ui-bg-surface: #2d2d2d;
  /* ... */
}
```

### Phase 3: 컴포넌트 적용

```css
/* src/components/Header/Header.module.css */
.header {
  background-color: var(--ui-bg-base);
  color: var(--ui-text-primary);
  border-bottom: 1px solid var(--ui-border-default);
}

.header button:focus {
  outline: 2px solid var(--ui-focus-ring);
}
```

### Phase 4: 쇼케이스 영역 스코핑

```css
/* 쇼케이스 영역은 기존 --ds-* 사용 */
[data-showcase] {
  background-color: var(--ds-color-bg-base);
  color: var(--ds-color-text-primary);
}
```

## E06과의 관계

| 에픽 | 범위 | 관심사 |
|------|------|--------|
| **E06 (Site Style)** | UX 철학 | "어떻게 안내할 것인가" (Minimal UI, 툴팁) |
| **E07 (UI Namespace)** | 토큰 구조 | "어디에 어떤 토큰을 쓸 것인가" |

E06의 Site Style은 **사용 패턴**을 정의하고, E07의 UI Namespace는 **토큰 적용 범위**를 정의한다.

---

## 참고

- Site UI는 **다크 계열 중립 톤** 권장 (쇼케이스와 시각적 구분)
- 쇼케이스 내부에서 서로 다른 테마를 동시에 표시 가능해야 함
- 접근성 기능은 Site UI와 쇼케이스 **모두에 영향**을 줄 수 있음 (별도 고려 필요)
