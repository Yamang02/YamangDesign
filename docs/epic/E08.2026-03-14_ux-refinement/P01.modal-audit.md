# P01: 모달 재평가

## 목표

전체 모달(GlobalSettings / SemanticMapping / ScaleSelection / ComponentDetail)의
UI 구조와 토큰 표시 당위성을 평가하고 개선한다.

## 구현 상세

### GlobalSettingsModal

**문제**: `STYLE_OPTIONS` 배열이 하드코딩되어 있어 `glassmorphism` 등 신규 프리셋 추가 시 누락 가능.

**개선**: `src/styles/presets/index.ts`의 preset registry에서 동적으로 옵션 파생.

```ts
// 변경 전
const STYLE_OPTIONS = [
  { value: 'minimal', label: 'Minimal' },
  { value: 'neumorphism', label: 'Neumorphism' },
  { value: 'brutalism', label: 'Brutalism' },
];

// 변경 후
const STYLE_OPTIONS = getStylePresetNames().map((name) => ({
  value: name,
  label: capitalize(name),
}));
```

### SemanticMappingModal

**문제**: 3패널 의도(ScaleGrid | ColorUsageDiagram | 컴포넌트 예시)에서 세 번째 패널이 미구현. 레이아웃이 의도와 실제 불일치.

**개선**: 2패널(좌: ScaleGrid, 우: ColorUsageDiagram)로 명시적 단순화. 불필요한 레이아웃 슬롯 제거.

### ScaleSelectionModal

**문제**: 직접 HEX 입력 옵션이 scale 기반 시맨틱 매핑 시스템과 개념적으로 충돌. "이 토큰은 어느 scale/step인가"라는 질문에 HEX 직접 입력은 답이 아님.

**개선**: 직접 HEX 입력 섹션 제거. scale + step 선택 UI에 집중.

### ComponentDetailModal

**사용처 감사**: 현재 실제로 어디서 열리는지 확인 필요.
- 사용처가 없으면 제거
- 사용처가 있으면 현행 유지

## 체크리스트

- [x] `GlobalSettingsModal` STYLE_OPTIONS를 preset registry 기반으로 교체
- [x] `SemanticMappingModal` 3패널 → 2패널 레이아웃 단순화 (이미 2패널, 주석 명시)
- [x] `ScaleSelectionModal` HEX 직접 입력 섹션 제거
- [x] `ComponentDetailModal` 사용처 grep 감사 (Build Atoms/Molecules/Organisms/Components에서 사용 → 유지)
- [x] 사용처 없으면 ComponentDetailModal 제거, 있으면 현행 유지
- [ ] 전체 모달 동작 확인
