# P04: Shell Overview 컴포넌트 인벤토리

## 목표

Shell > Overview를 추상 다이어그램에서 실제 Shell 구성 컴포넌트 인벤토리로 교체한다.
"Shell은 어떤 컴포넌트로 이루어져 있는가"가 한눈에 드러나야 한다.

## 구현 상세

### 현재 문제

Overview 섹션이 Header/Nav/Footer 영역 박스 + 토큰 태그 나열에 그쳐
실제 Shell 컴포넌트(Header, HeaderNav, Navigation, NavDropdown, Footer)가 전혀 보이지 않음.
Token Map 섹션과 내용이 중복되는 부분도 있음.

### 컴포넌트 인벤토리 카드 구조

각 Shell 컴포넌트를 카드로 표시:

```
┌─────────────────────────────────┐
│ Header                          │
│ src/components/Header           │
│ ─────────────────────────────── │
│ --shell-bg-base                 │
│ --shell-text-primary            │
└─────────────────────────────────┘
```

| 컴포넌트 | 담당 토큰 |
|---------|---------|
| Header | `--shell-bg-base`, `--shell-text-primary` |
| HeaderNav | `--shell-bg-hover`, `--shell-text-hover`, `--shell-border-focus` |
| Navigation | `--shell-bg-hover`, `--shell-action-primary` |
| NavDropdown | `--shell-bg-surface`, `--shell-border-subtle` |
| Footer | `--shell-bg-subtle`, `--shell-text-secondary` |

### Live Preview 연동

카드 클릭 시 Live Preview 섹션으로 스크롤 + 해당 컴포넌트 하이라이트.
(구현 복잡도에 따라 스크롤만으로 제한 가능)

### Token Map 섹션 조정

현재 Token Map과 Overview의 내용 중복 해소:
- Overview = 컴포넌트 인벤토리 (어떤 컴포넌트가 있는가)
- Token Map = 토큰 상세 (어떤 토큰이 어느 값인가)
