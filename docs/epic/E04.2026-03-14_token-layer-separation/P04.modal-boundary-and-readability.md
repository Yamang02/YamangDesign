# P04: 모달 경계 명시 및 가독성 표준화

## 목표

혼재 모달의 크롬/콘텐츠 경계를 CSS 수준에서 명시하고, 모든 CSS 모듈 파일에 컨텍스트 선언 주석을 표준화한다.

## 구현 상세

### 1. 모달 크롬/콘텐츠 경계 명시

`ScaleSelectionModal`, `SemanticMappingModal`, `ComponentDetailModal`은 단일 `[data-shell]` 래퍼 안에 크롬과 콘텐츠가 공존한다. CSS를 섹션으로 명확히 분리한다.

```css
/* ─── Chrome (shell) ─────────────────────────────── */
.modalOverlay { ... }          /* --shell-overlay */
.modal { ... }                 /* --shell-bg-base */
.modalHeader { ... }           /* --shell-border-default */
.modalFooter { ... }

/* ─── Content (ds — 테마 반응형) ─────────────────── */
.paletteGrid { ... }           /* --ds-color-primary-* */
.scaleStep { ... }
.swatchLabel { ... }
```

콘텐츠 영역에는 `data-context="content"` 또는 wrapper 클래스로 `--ds-*` 복원이 필요한 경우 명시.

### 2. CSS 파일 헤더 주석 표준화

모든 CSS 모듈 파일 상단에 컨텍스트 선언 추가:

```css
/* @layer shell — [data-shell] 컨텍스트. 색상 토큰은 --shell-* 직접 참조. */
/* @layer ds — 테마 반응형. 모든 토큰 --ds-* 사용. */
/* @layer mixed — 크롬(--shell-*)과 콘텐츠(--ds-*) 혼재. 섹션 주석 참조. */
```

**대상 파일:**
- shell 영역 CSS 모듈 전체 → `@layer shell`
- ds 영역 CSS 모듈 전체 → `@layer ds`
- ScaleSelectionModal, SemanticMappingModal, ComponentDetailModal → `@layer mixed`

### 3. LabLayout preview 복원 블록 주석 보강

```css
/* ─── Preview 복원 ([data-context="preview"]) ───────
   [data-shell] 안에서 테마 토큰을 복원.
   ThemeProvider가 주입한 *-global 별칭을 참조.
   이 블록 내부는 --ds-* 유지 필수. ───────────────── */
[data-shell] [data-context="preview"] { ... }
```

### 4. 컴포넌트 페이지 모달(ComponentDetailModal) 세분화

컴포넌트 쇼케이스 모달에서는 **섹션 표시·라벨은 shell**, **실제 컴포넌트 요소만 ds** 로 구분한다.

**원칙:**
- 모달 body는 `data-context="preview"`를 붙이지 않음 → 기본이 shell.
- 콘텐츠 측에서 **실제 컴포넌트만** `data-context="preview"`로 감싼다.

**Shell로 취급 (Components.module.css 등):**
- 섹션 제목: `.showcaseModalSectionTitle` → `--shell-text-primary`
- 그룹 라벨: `.showcaseModalGroupLabel` → `--shell-text-secondary`
- Variant 셀 라벨: `.showcaseVariantItemLabel` → `--shell-text-muted`
- Design tokens 섹션: `.tokenGroupTitle`, `.tokenName`, `.tokenLabel`, `.tokenRow`, `.tokenValue`, `.tokenSwatch` → `--shell-*`
- 섹션 구분선/테두리: `--shell-border-subtle` 등

**Ds로 취급 (테마 반응형):**
- `VariantCell` 내부: 라벨은 그대로 두고, **자식(Button, Badge, Card 등)만** `<div data-context="preview" className={styles.showcaseVariantPreview}>` 로 래핑.
- form-example: 폼 전체를 감싼 루트 `div`에 `data-context="preview"` → Card/Input/Button에 테마 적용.

**구현 위치:**
- `ComponentDetailModal.tsx`: body에서 `data-context="preview"` 제거.
- `Components.tsx`: `VariantCell`에 preview 래퍼 추가, form-example 루트에 `data-context="preview"`.
- `Components.module.css`: 모달 내 섹션/라벨/토큰 테이블 스타일을 `--shell-*`로 통일, `.showcaseVariantPreview` 추가.
