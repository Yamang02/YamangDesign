# P02: Sys 토큰 분리

## 목표

`--ds-color-system-*`를 `--sys-*` 네임스페이스로 분리하여 시스템/상태 컬러가 테마 반응형 컬러와 명확히 구분되도록 한다.

## 구현 상세

**새 파일 생성: `src/styles/sys-variables.css`**

error, warning, success, info 계열 색상을 정의. 테마 무관, 어디서나 고정값 사용. 기존 `-bg` 사용처는 `-subtle`로 통일.

```css
/* sys-variables.css */
/* 시스템 상태 컬러. 테마 무관, 컨텍스트 무관. */
:root {
  --sys-color-error:   #dc2626;
  --sys-color-error-subtle:   #fef2f2;
  --sys-color-warning: #d97706;
  --sys-color-warning-subtle: #fffbeb;
  --sys-color-success: #16a34a;
  --sys-color-success-subtle: #f0fdf4;
  --sys-color-info:    #2563eb;
  --sys-color-info-subtle:    #eff6ff;
}
```

(기존 `--ds-color-system-*-bg` 참조는 `--sys-color-*-subtle`로 교체. `-emphasis` 사용처가 있으면 동일 톤 강조색으로 `--sys-*`에 추가할지 결정.)

**기존 토큰 제거 및 참조 업데이트:**
- `shell-variables.css`(구 `ui-variables.css`)에서 `--ui-badge-*` 등 시스템 성격 토큰 검토
- `--ds-color-system-error` 등 기존 참조를 `--sys-color-error` 등으로 일괄 교체
- `src/styles/index.ts`에 `sys-variables.css` import 추가
- `ThemeProvider.tsx`에서 system 색상 주입 로직이 있다면 `--sys-*`로 정리

**네이밍 매핑 (기존 → 신규):**
- `--ds-color-system-{name}-bg` → `--sys-color-{name}-subtle` (P02 예시와 통일)
- `--ds-color-system-{name}` → `--sys-color-{name}`
- `--ds-color-system-{name}-emphasis` → 유지 시 `--sys-color-{name}-emphasis`, 또는 제거 후 필요 시 subtle/기본만 사용
