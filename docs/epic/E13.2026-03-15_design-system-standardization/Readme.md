# Epic E13: Design System Standardization

## 목표
프로젝트 전반에 산재된 UI 불일치를 제거하고, 컴포넌트·스타일·파일 구조를 일관된 기준으로 정비한다.
랩 페이지 섹션 시스템, 컴포넌트 API, 토큰 커버리지, 파일 구조를 순차적으로 표준화해
신규 기능 추가 시 "무엇을 어떻게 써야 하나"를 고민하지 않아도 되는 상태를 만든다.

## 배경 / 맥락
E11(도메인 모델), E12(src 3계층 구조) 이후 코드 아키텍처는 정비되었으나,
아래 영역은 기준 없이 누적된 상태:
- 랩 페이지의 비교 섹션·프리뷰·메뉴가 랩마다 다르게 구현됨
- CSS에 토큰 없이 하드코딩된 값 다수 존재
- 컴포넌트 이벤트 핸들러 이름, open/close 패턴, aria 속성이 컴포넌트마다 다름
- 타입 파일 위치, index.ts export 방식이 혼재

## 특이점
- 각 Phase는 독립적으로 실행 가능하나, P01 → P02 → P03 → P04 순서 권장
- P01(파일 구조)은 이후 작업에서 타입 수정 시 기반이 되므로 먼저 처리
- P03(컴포넌트 API)은 브레이킹 체인지가 없도록 점진 적용
- P04(랩 섹션 시스템)은 실제 UX 레이어이므로 P01~P03 이후에 안정적으로 진행
