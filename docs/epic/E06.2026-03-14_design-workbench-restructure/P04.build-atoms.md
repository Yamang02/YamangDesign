# P04: Build — Atoms 정비

## 목표

기존 Components 페이지를 "Atoms" 컨텍스트로 재포지셔닝한다.
컴포넌트가 Atomic Design 계층에서 어디에 속하는지 명시하고,
각 컴포넌트 카드에서 "이 Atom이 어떤 Molecules에서 쓰이는가"를 연결한다.

## 구현 상세

### 페이지 재포지셔닝

라우팅: `/components` → `/build/atoms`
컴포넌트 파일: 기존 `Components.tsx`를 `Atoms.tsx`로 복사/이동하거나,
App.tsx에서 동일 컴포넌트를 새 경로에 매핑.

페이지 타이틀: `"Components"` → `"Atoms"`
LabLayout 부제 추가: `"단일 UI 요소 — 더 이상 분해할 수 없는 기본 단위"`

### Atoms 목록 확정

현재 Components 그리드에 있는 것들이 모두 Atom이므로 그대로 유지:
- Button / Input / Select / Badge / Icon / Avatar / Card / Tooltip

Card는 엄밀히 Molecule이지만 이 프로젝트에서는 컨테이너 Atom으로 취급.

### ComponentCard에 계층 표시 추가

각 ComponentCard에 "어떤 Molecule에서 쓰이는가" 힌트 배지 추가 (P05 이후 연결):

```tsx
// ComponentCard props 확장
interface ComponentCardProps {
  ...
  usedIn?: string[];  // ['FormField', 'SearchBar']
}
```

현재는 `usedIn` 비어있어도 되고, P05에서 Molecules가 구현될 때 채워진다.

### FontLab Detail 개선 흡수

E05 P03(FontLab Detail JSON → 구조화)을 여기서 같이 진행해도 되지만,
FontLab은 Labs 카테고리이므로 별도 소규모 작업으로 처리. 이 Phase 범위 외.

영향 범위:
- `src/App.tsx` 또는 라우팅 파일 (경로 변경)
- `src/pages/build/Atoms/` 디렉토리 신규 생성 (기존 Components 내용 이동)
- `src/components/ComponentCard/ComponentCard.tsx` (`usedIn` prop 추가)
