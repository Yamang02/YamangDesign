# P03: 상수·타입 네이밍 정리

## 목표

P02 이후에도 소스 내에 남는 `layouts` 문자열 참조 및 dead export를 제거하여
코드와 UI 용어를 완전히 일치시킨다.

## 구현 상세

### 점검 항목

- `src/constants/` 내 dead export (더 이상 사용되지 않는 상수)
- `src/config/nav-categories.ts` 내 카테고리 키/레이블이 실제 네비게이션과 일치하는지 확인
- 타입 정의에서 `layouts` / `showcase` 문자열이 남아있는 경우 제거
- `src/components/index.ts` 미사용 re-export 감사

### 기준

- 실제로 import되지 않는 export는 제거
- 이름이 UI 레이블과 다른 경우 UI 레이블 기준으로 통일
- 타입 변경 시 TypeScript 오류 없음을 빌드로 확인
