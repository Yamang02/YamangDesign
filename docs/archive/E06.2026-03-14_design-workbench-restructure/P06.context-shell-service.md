# P06: Context — Shell & Service 뷰

## 목표

"이 디자인 시스템이 실제로 어디에 쓰이는가"를 맥락별로 분리하여 보여준다.
- **Shell**: 이 도구 자체의 크롬 — `--shell-*` 토큰이 어떻게 렌더링되는지
- **Service**: DS 테마가 적용된 서비스 UI — 실제 사용자가 보는 화면

## 구현 상세

### Context/Shell 페이지

앱 크롬(Header/Navigation/Footer) 자체를 **디자인 아티팩트로** 시각화한다.
E05 P01(Shell 중립화) 이후의 실제 크롬 렌더링을 "설명이 붙은 해부도"처럼 보여준다.

**섹션 구성:**

```
Shell Context
├── Overview       크롬 구조 다이어그램 (영역별 --shell-* 토큰 오버레이)
├── Token Map      크롬 각 영역이 어떤 --shell-* 토큰을 쓰는지 표
└── Live Preview   실제 Header + Nav + Footer 렌더 (축소 프레임)
```

**Token Map 표:**
```
영역                  토큰
Header 배경           --shell-bg-base
Header 텍스트         --shell-text-primary
Nav 호버 배경         --shell-bg-hover
Nav 호버 텍스트       --shell-text-hover
Nav 활성 테두리       --shell-border-focus
ComparisonCard 선택   --shell-action-primary (border)
버튼 배경             --shell-action-primary
포커스 링             --shell-focus-ring
```

**Live Preview:**
실제 `<Header>` + `<Navigation>` 컴포넌트를 작은 프레임 안에 렌더.
`[data-shell]` 컨텍스트가 이미 적용되어 있으므로 별도 조작 없이 표시됨.

### Context/Service 페이지

현재 Playground의 "Brand" 컬럼을 확장한 전체 서비스 UI 미리보기.
Playground와 차이: Playground는 "조합 실험 도구", Service는 "결과물 미리보기".

**섹션 구성:**
```
Service Context
├── Controls       Palette / Style / Font 선택 (Playground와 동일)
├── Page Preview   Landing 스타일 레이아웃 (Hero + CardGrid + CTA)
└── Component Set  현재 테마의 전체 컴포넌트 렌더 (Atoms → Organisms 순)
```

Page Preview는 실제 서비스 페이지처럼 보이는 목업:
```
[Hero section]   큰 타이틀 + 부제 + CTA Button
[Card Grid]      ActionCard × 3
[Form section]   FormCard
[Footer]         링크 + 저작권
```
이 전체가 `data-context="preview"` 래핑 + 선택된 `themeVars` 적용.

### 기존 Pages (Landing/Dashboard/Card Grid) 처리

기존 `/pages/*` 경로를 `/context/service`의 탭/섹션으로 통합하거나
별도 "Examples" 탭으로 유지. 즉각 제거하지 않고 점진적으로 흡수.

영향 범위:
- `src/pages/context/Shell/ShellContext.tsx` (신규)
- `src/pages/context/Service/ServiceContext.tsx` (신규, Playground 참조)
- `src/pages/context/Service/ServiceContext.module.css`
- `src/pages/context/Shell/ShellContext.module.css`
