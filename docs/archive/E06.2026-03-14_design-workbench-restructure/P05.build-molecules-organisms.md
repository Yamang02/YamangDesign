# P05: Build — Molecules & Organisms

## 목표

Atoms를 조합해 의미 있는 UI 패턴을 만드는 Molecules 계층과,
여러 Molecules를 묶어 페이지 섹션을 구성하는 Organisms 계층을 구현한다.
각 레벨에서 "어떤 Atoms/Molecules가 조합되었는가"를 명시한다.

## 구현 상세

### Molecules 페이지

**포함할 패턴 (1차):**

| Molecule | 구성 Atoms | 설명 |
|----------|-----------|------|
| FormField | Input + Label + helper text | 레이블·에러·힌트가 붙은 입력 필드 |
| SearchBar | Input + Button(Icon) | 검색 입력 + 실행 버튼 |
| ProfileCard | Avatar + Badge + text | 사용자 식별 카드 |
| ActionCard | Card + Button | 행동 유도 카드 (CTA) |
| TagGroup | Badge × n | 복수 태그 묶음 |

각 Molecule은 Atoms 페이지와 같은 `ComponentCard` 패턴으로 표시:
- 카드 프리뷰: Molecule 렌더
- 카드 하단: 구성 Atoms 목록 (아이콘 + 이름)
- 클릭 시 모달: Variants + 사용 코드 예시 + 토큰 목록

**Atoms 역참조 연결:**
P04에서 추가한 `usedIn` prop을 이 Phase에서 채워 넣는다.
```typescript
// Atoms 페이지의 VARIANT_COUNTS/SHOWCASE_GRID_IDS와 연동
const ATOM_USED_IN: Record<ShowcaseSectionId, string[]> = {
  input:  ['FormField', 'SearchBar'],
  button: ['SearchBar', 'ActionCard'],
  badge:  ['ProfileCard', 'TagGroup'],
  avatar: ['ProfileCard'],
  card:   ['ActionCard'],
  ...
};
```

### Organisms 페이지

**포함할 패턴 (1차):**

| Organism | 구성 Molecules/Atoms | 설명 |
|----------|---------------------|------|
| HeaderBar | Navigation + ProfileCard + Button | 앱 상단 바 |
| SidebarNav | NavItem × n + Badge | 좌측 내비게이션 |
| CardGrid | ActionCard × n | 카드 그리드 레이아웃 |
| FormCard | FormField × n + Button | 폼 + 카드 래퍼 |
| StatusBanner | Icon + sys-color text | 시스템 상태 알림 |

Organisms는 실제 페이지에서 사용하는 컴포지션이므로,
각 카드 프리뷰를 실제 크기에 가깝게(full-width 또는 large card) 렌더.

### 공통 패턴 — Composition Map

각 Molecule/Organism 모달 상단에 "조합 구조" 텍스트 맵:

```
FormCard
└── Card
    └── FormField × 2
        ├── Input
        └── Label
    └── Button
```

이를 `CompositionMap` 컴포넌트(들여쓰기 트리 형태)로 구현.
