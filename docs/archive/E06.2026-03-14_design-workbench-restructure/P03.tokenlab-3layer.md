# P03: Token Lab 3-레이어 뷰

## 목표

Token Lab을 `--shell-*` / `--ds-*` / `--sys-*` 세 레이어를 명확히 구분한 구조로 재설계한다.
각 섹션이 해당 레이어의 live 값을 보여주며, 레이어 간 매핑 관계도 시각화한다.

## 구현 상세

### 새 TOC 구조

```
Token Lab
├── Architecture      3레이어 관계 다이어그램
├── Shell Tokens      --shell-* live 값 (고정 라이트 크롬)
├── DS Tokens         --ds-* live 값 (테마 반응형)
│     Global         원시 토큰 (color scale, spacing, typography, motion)
│     Alias          의미 토큰 (bg, text, border, action)
└── Sys Tokens        --sys-* live 값 (상태 색상 고정)
```

### Architecture 섹션 (신규)

세 레이어의 관계를 텍스트 다이어그램으로 표시:

```
[data-shell] 컨텍스트
  --shell-bg-base ──────→ --ds-color-bg-base (오버라이드)
  --shell-text-primary ─→ --ds-color-text-primary
  --shell-shadow-* ─────→ --ds-shadow-*
  (고정 라이트, 테마 무관)

:root DS 컨텍스트 (ThemeProvider 주입)
  --ds-color-bg-base ←── 팔레트 × 스타일 조합
  --ds-shadow-* ←──────  스타일 정의
  (테마 반응형)

[어디서나] Sys 토큰
  --sys-color-error ────  고정값, 컨텍스트 무관
  --sys-color-warning
  (상태 색상, 테마 무관)
```

이를 `LayerDiagram` 컴포넌트(SVG 또는 CSS flex)로 구현.
클릭 시 해당 섹션으로 스크롤.

### Shell Tokens 섹션 (신규)

`shell-variables.css`의 `:root` 토큰 목록을 카테고리별로 표시:

```
배경 (bg)       --shell-bg-base, surface, muted, elevated, hover
텍스트 (text)   --shell-text-primary, secondary, tertiary, on-action, hover
경계선 (border) --shell-border-default, subtle, strong, hover, focus
액션 (action)   --shell-action-primary, hover, active
포커스          --shell-focus-ring, focus-glow
그림자 (shadow) --shell-shadow-sm, md, lg
오버레이        --shell-overlay, tooltip-bg, tooltip-text
배지 (badge)    --shell-badge-custom, semantic, natural
```

각 토큰은 `ColorTokenCard` (색상) 또는 `ScalarTokenCard` (그 외)로 표시.
E05 P04에서 제안한 `ScalarTokenCard`를 여기서도 사용.

### DS Tokens 섹션 (기존 Global/Alias 재구성)

기존 `GlobalSection` + `AliasSection`을 유지하되:
- Global에 Spacing / Typography / Motion 카테고리 추가 (E05 P04 내용 흡수)
- Alias에 `--ds-color-bg-*`, `--ds-color-text-*`, `--ds-color-border-*`, `--ds-color-action-*` 전체 표시

### Sys Tokens 섹션 (신규)

`sys-variables.css`의 토큰 목록:

```
Error    --sys-color-error, error-subtle, error-emphasis
Warning  --sys-color-warning, warning-subtle, warning-emphasis
Success  --sys-color-success, success-subtle, success-emphasis
Info     --sys-color-info, info-subtle, info-emphasis
```

각 그룹을 `SystemColorGroup` 컴포넌트로:
- base 색상 큰 스와치
- subtle / emphasis 작은 스와치
- 실제 사용 예시: Input 에러 상태, 배지 등

### Component Inspector 개선

기존 `ComparisonCard` → `useTheme()` + `getThemeVariables(palette, style)`로 `styleVars` 주입하여 테마 반응형으로 전환.

영향 범위:
- `src/pages/layouts/TokenLab/TokenLab.tsx` (전체 재구성)
- `src/pages/layouts/TokenLab/TokenLab.module.css`
