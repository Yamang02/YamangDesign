# P05: Token Lab UI 정렬 (테이블 구조)

## 목표

Token Lab의 토큰 표시 방식을 다른 랩 및 Build의 Design tokens 패턴과 맞추고,
토큰 목록을 **테이블 구조**로 통일하여 가독성과 일관성을 높인다.

## 배경

- Token Lab만 카드 그리드(`ColorTokenCard` / `ScalarTokenCard` / `SystemColorGroup`)가 섹션별로 제각각이라 Palette Lab, Style Lab, Font Lab과 이질적으로 보임.
- Build > 컴포넌트 상세 모달의 "Design tokens"는 이미 **테이블**(토큰 이름 · 라벨 · 값 · 색상 시 스와치) 구조를 사용하고 있어, 같은 정보 유형에 다른 레이아웃이 적용된 상태.
- 테이블로 통일하면 스캔하기 쉽고, 다른 랩의 `LabSection` + 표 형식 콘텐츠 패턴과 정렬된다.

## 구현 방향

### 공통 토큰 테이블

- **열**: 토큰 이름(`--ds-*` 등) | 값(현재 computed) | 색상 토큰이면 스와치(선택).
- Build의 `ComponentDetail` 내 `tokenTable` / `tokenGroup` 패턴을 참고하여, Token Lab 전용 **공통 `TokenTable`**(또는 동일 스타일 클래스) 컴포넌트/마크업 사용.
- 카테고리(Shell 배경/텍스트/경계선, DS Global Color/Spacing/Typography/Motion, DS Alias, Sys Error/Warning 등)는 **섹션 제목 + 하나의 테이블** 또는 **카테고리별 서브테이블**로 구성.

### 적용 범위

| 섹션 | 변경 내용 |
|------|-----------|
| Shell Tokens | 카테고리별 카드 그리드 → 카테고리별 테이블(토큰 | 값 | 스와치) |
| DS Global | Color/Spacing/Typography/Motion 각각 그리드 → 각각 테이블 |
| DS Alias | ColorTokenCard 그리드 → 단일 테이블(또는 역할 그룹별 서브테이블) |
| Sys Tokens | SystemColorGroup 블록 → 테이블(토큰 | 값 | 스와치), 사용 예시는 유지 가능 |
| Component Inspector | "사용 토큰 목록"을 리스트 → 테이블 행 구조로 정리 (선택) |
| Architecture | 유지. 필요 시 `LabSection` 스타일과만 정리 |

### 다른 랩과의 일관성

- 모든 토큰 섹션에 `LabSection` 사용 유지.
- 테이블은 `data-context="preview"` 등 기존 토큰 반영 규칙을 그대로 적용.
- Shell/DS/Sys 구분(아키텍처)은 TOC와 섹션 제목으로 유지.
