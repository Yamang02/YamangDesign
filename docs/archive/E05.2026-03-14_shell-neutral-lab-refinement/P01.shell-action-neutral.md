# P01: Shell 액션/호버 토큰 중립화

## 목표

`shell-variables.css`의 브랜드 컬러 하드코딩을 제거하고,
셸 크롬이 어떤 팔레트 선택과 무관하게 항상 동일한 중립 회색 계열로 표시되도록 한다.
단, 호버 하이라이트는 브랜드와 어울리는 warm slate를 유지한다.

## 구현 상세

### `[data-shell]` 매핑 확인

`[data-shell]` 블록의 DS 오버라이드는 `--shell-*` 값을 참조하므로,
`:root` 값 변경만으로 자동 반영된다. 별도 수정 불필요.

변경 영향 범위 (모두 `:root`의 `--shell-*` 참조이므로 CSS 값만 변경 시 자동 반영):

**문서 기재:**
- Header / HeaderNav — 활성 탭, 포커스 링
- Navigation / NavDropdown — hover/active 상태
- Footer — 링크 hover
- GlobalSettingsModal — 버튼 스타일
- LabLayout — 탭바 활성 항목, ComparisonCard selected 테두리

**추가 시각 검증 권장 (동일 토큰 참조):**
- **PaletteLab**: ScaleStepGrid(포커스 링), ScaleSelectionModal·ScaleSelectorPanel(버튼/포커스), ThemeSearchBar(포커스), ThemeTabNavigation(활성 탭), SemanticMappingModal(버튼), ColorUsageDiagram(선택 상태)
- **Components 쇼케이스**: `Components.module.css` — 포커스/테두리
- **ComponentDetailModal** / **DetailPanel** — border-default
- **LabOverview** — border-subtle
- **FontOverviewDiagram** / **StyleOverviewDiagram** — border-subtle (구분선만, 색상 변화 작음)
