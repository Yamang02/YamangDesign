# P02: StyleLab Shadow 데모 수정

## 목표

StyleLab의 Shadow Comparison 섹션에서 shadow demo 박스가
스타일 변경에 반응하도록 수정한다.

## 구현 상세

### 현재 문제

`StyleLab.tsx`의 Shadow Comparison 섹션:

```tsx
// 현재: --shell-shadow-* 고정값 → 스타일 변경 시 shadow 안 바뀜
<div
  className={styles.shadowDemo}
  style={{
    boxShadow: `var(--shell-shadow-${size})`,
    backgroundColor: 'var(--shell-bg-base)',
  }}
>
```

`ComparisonCard`의 `styleVars` prop으로 각 스타일의 DS 변수가 주입되는데,
데모 박스가 `--shell-shadow-*`(고정)를 참조하므로 스타일별 차이가 표시되지 않는다.

### 수정 방향

데모 박스를 `data-context="preview"` 내부로 이동하고 `var(--ds-shadow-*)` 참조로 변경.
`styleVars`로 주입된 값이 이 컨텍스트 안에서 `--ds-shadow-*`를 오버라이드하므로
스타일별 shadow가 정확히 표시된다.

```tsx
// 변경 후: --ds-shadow-* 참조 → styleVars 주입값 반영
<div data-context="preview">
  {shadowKeys.map((size) => (
    <div
      key={size}
      className={styles.shadowDemo}
      style={{
        boxShadow: `var(--ds-shadow-${size})`,
        backgroundColor: 'var(--ds-color-bg-base)',
      }}
    >
      shadow-{size}
    </div>
  ))}
</div>
```

### bgColor 처리

현재 `const bgColor = '#f5f5f5'` 하드코딩이 `createStyle(styleDef, bgColor)` 및
각 ComparisonCard의 래퍼 `backgroundColor`에 쓰임.
`getStyleVariables(styleName, bgColor)`가 배경색을 포함한 DS 변수를 반환하므로,
demo 박스 래퍼의 `backgroundColor`는 `var(--ds-color-bg-base)`로 교체하고
하드코딩 `bgColor` 상수는 `getStyleVariables` 호출에만 남긴다.

영향 범위: `StyleLab.tsx`, `StyleLab.module.css`

## 체크리스트

- [ ] Shadow Comparison 섹션 demo 박스를 `data-context="preview"` 래핑
- [ ] `var(--shell-shadow-*)` → `var(--ds-shadow-*)` 교체
- [ ] `backgroundColor: bgColor` → `backgroundColor: 'var(--ds-color-bg-base)'` 교체
- [ ] Component Comparison 섹션 — 이미 `data-context` 없이 ds 토큰 참조 중이므로 동일 패턴 확인
- [ ] minimal / neumorphism / brutalism 세 스타일의 shadow가 시각적으로 다르게 표시되는지 확인
