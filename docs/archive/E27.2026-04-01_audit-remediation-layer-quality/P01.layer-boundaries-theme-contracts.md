# P01: 레이어 경계·테마 계약

## 목표

- `domain` → `@app` **역방향 import가 제거**되거나, 앱 레이어 **브리지**로 흡수되어 AR-01-02 엄격 방향과 모순되지 않게 정리된다.
- `src/shared/@types/theme.d.ts`의 **`@domain` 의존이 제거**되고, `PaletteName` 등은 `shared` 또는 전용 **contracts** 모듈에서 일관되게 참조된다.
- 변경 후 **`tsc`·기존 테스트·빌드**가 통과한다.

## 구현 상세

### 접근

1. **PaletteName / PaletteSelection:** `src/shared/types/palette-name.ts`, `palette-selection.ts`에 두고 domain·app이 참조. `theme-presets`는 `@shared`에서 타입만 가져온다.
2. **ThemeProvider:** `src/app/providers/ThemeProvider.tsx`로 이동 — UI·저장소·훅(`@app`) 의존을 앱 레이어에 둔다. `domain/themes/index.ts`는 `useTheme`·`combineTheme`·`ThemeContext`만 export.
3. **TokenLab categories.json:** `src/shared/content/labs/token-lab/categories.json`으로 옮겨 domain 테스트가 `@app` JSON에 의존하지 않게 함.

### 변경 범위 (실제)

- `src/shared/types/palette-name.ts`, `palette-selection.ts`, `shared/@types/theme.d.ts`
- `src/domain/constants/theme-presets.ts`, `domain/palettes/types.ts`, `domain/themes/ThemeContext.ts`, `domain/themes/index.ts`
- `src/app/providers/ThemeProvider.tsx`, `App.tsx`
- `src/shared/content/labs/token-lab/categories.json` (삭제: `app/content/labs/token-lab/categories.json`)
