# P06: 컴포넌트-값 매핑 탭 신설

## 목표

P05에서 만든 설정 페이지의 세 번째 탭 **"컴포넌트 매핑"**을 구현한다. 각 UI 컴포넌트(Button, Input, Card 등)가 어떤 토큰 값을 사용하는지 직접 편집하고 미리보는 기능을 제공한다.

## 배경 / 동기

현재 워크벤치에서는:
- 팔레트 · 시맨틱 토큰 계층은 설정 가능하나
- **"어떤 컴포넌트가 어떤 토큰을 쓰는가"** 를 시각적으로 탐구하거나 직접 오버라이드하는 수단이 없다

컴포넌트-값 매핑 탭은 이 계층을 명시적으로 드러내고, 매핑 편집 → Live Preview 연동까지 제공하는 것을 목표로 한다.

## 구현 상세

### 1. 탭 레이아웃

3-패널 구성:
```
[컴포넌트 선택 패널] | [매핑 편집 패널] | [Live Preview 패널]
```

- **좌**: 컴포넌트 목록 (Button / Input / Select / Card / Badge 등 / 카테고리별 그룹)
- **중**: 선택된 컴포넌트의 토큰 매핑 테이블
  - 속성명 (bg, text, border, radius, shadow 등)
  - 현재 적용 토큰 값 (별칭 표시 + 실제 hex/값)
  - 오버라이드 입력 (토큰 선택기 또는 직접 값 입력)
- **우**: 선택된 컴포넌트 Live Preview (현재 매핑 기준 렌더)

### 2. 데이터 구조

컴포넌트 매핑 정의는 JSON으로 관리 (E09 데이터 분리 원칙):

```
src/content/labs/design-settings/
  component-mapping.json   # 컴포넌트별 사용 토큰 목록 + 기본 매핑
```

스키마 예시:
```json
{
  "components": [
    {
      "id": "button",
      "label": "Button",
      "category": "action",
      "tokens": [
        { "prop": "background", "default": "--ds-action-bg", "description": "기본 배경" },
        { "prop": "color", "default": "--ds-action-text", "description": "텍스트 색상" }
      ]
    }
  ]
}
```

### 3. 오버라이드 저장

- 컴포넌트 매핑 오버라이드는 `localStorage` 또는 `useTheme` 컨텍스트에 저장
- 프리셋 저장 시 컴포넌트 매핑 오버라이드 포함 여부 결정 (선택)
- 내보내기 / 가져오기: 기존 yamang JSON 포맷에 `componentMapping` 키로 통합

### 4. Live Preview

- 현재 `Build` 페이지의 컴포넌트 렌더 방식 참조
- 중 패널에서 토큰 값 변경 → 우 패널 즉시 반영
- 여러 variant(default / hover / disabled 등) 동시 표시 (선택)

### 5. 범위 결정 사항 (구현 전 확정)

- 오버라이드 적용 범위: 전역 vs 프리셋 단위
- 저장 방식: localStorage 직접 vs useTheme 통합
- 지원 컴포넌트 초기 목록 확정 (권장: Button / Input / Select / Card / Badge)
