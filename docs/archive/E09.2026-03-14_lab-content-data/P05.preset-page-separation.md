# P05: StoredSettings v2 + 통합 설정 페이지 신설 + 두 모달 제거

## 목표

1. **데이터 모델 v2**: 프리셋이 전체 디자인 파이프라인 상태를 저장하도록 `StoredSettings` 스키마를 확장하고, 기존 v1 프리셋을 기본값으로 채워 마이그레이션한다.
2. **통합 설정 페이지**: `GlobalSettingsModal`과 `SemanticMappingModal`을 제거하고, 탭 기반 전용 페이지로 통합한다.

## 데이터 모델 v2

### 스키마

```ts
// v1 (현재)
interface StoredSettings {
  version: '1.0';
  palette: ExternalPalette;
  styleName: StyleName;
  systemPreset: SystemPresetName;
}

// v2 (목표)
interface StoredSettings {
  version: '2.0';
  palette: ExternalPalette;              // Layer 1: 컬러 (기본값: DEFAULT_PALETTE)
  palettePresetId?: string;             // Layer 1: 빌트인 프리셋 참조 (없으면 custom)
  semanticMapping: Partial<SemanticMapping> | null; // Layer 2: 시맨틱 오버라이드 (기본값: null)
  styleName: StyleName;                 // Layer 3: 스타일 (기본값: 'minimal')
  systemPreset: SystemPresetName;       // Layer 3: 시스템 색상 (기본값: 'default')
  // typography?: TypographySettings;   // Layer 4: 미래 확장
  // componentMapping?: ComponentOverrides; // Layer 5: 미래 확장
}
```

### v1 → v2 마이그레이션

v1 프리셋을 로드할 때 누락 필드를 기본값으로 채운다. 데이터 손실 없음.

```ts
function migrateV1ToV2(v1: StoredSettingsV1): StoredSettings {
  return {
    version: '2.0',
    palette: v1.palette,              // 그대로 유지
    palettePresetId: undefined,       // custom으로 간주
    semanticMapping: null,            // 기본 매핑 사용
    styleName: v1.styleName,          // 그대로 유지
    systemPreset: v1.systemPreset,    // 그대로 유지
  };
}
```

localStorage에 저장된 기존 설정 및 유저 프리셋 모두 이 함수를 거쳐 로드한다.

### 데이터 흐름

```
StoredSettings (v2)
  │
  ├─ palette / palettePresetId  →  createPalette()  →  컬러 스케일
  │                                      ↓ + semanticMapping
  │                               시맨틱 토큰 (--ds-bg-base, --ds-text-primary ...)
  │                                      ↓ + styleName / systemPreset
  │                               스타일 토큰 (--ds-surface-shadow, border-width ...)
  │                                      ↓ (미래) typography, componentMapping
  └─────────────────────────────  CSS 변수 주입 → 전체 워크벤치 반영
```

## 통합 설정 페이지

### 탭 구성

```
[프리셋] [시맨틱 매핑] [컴포넌트 매핑]
```

| 탭 | 편집 대상 | v2 키 |
|---|---|---|
| 프리셋 | 팔레트 편집 + 빌트인 프리셋 브라우저 + Style/System | `palette`, `palettePresetId`, `styleName`, `systemPreset` |
| 시맨틱 매핑 | 팔레트 스케일 → 시맨틱 역할 오버라이드 | `semanticMapping` |
| 컴포넌트 매핑 | 시맨틱 토큰 → 컴포넌트 속성 (P06 구현) | 슬롯만 확보 |

### 페이지 동작 원칙

- 페이지 열기 → 현재 활성 설정을 하나의 **draft** 객체로 로드
- 탭 전환해도 draft 유지 (탭 간 상태 공유)
- 시맨틱 매핑 탭은 항상 프리셋 탭의 팔레트를 컨텍스트로 사용 (별도 팔레트 선택 UI 없음)
- "적용" → draft → ThemeProvider 반영
- "프리셋으로 저장" → draft 전체 스냅샷 → `StoredPreset`
