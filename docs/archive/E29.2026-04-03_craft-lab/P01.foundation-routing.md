# E29 P01: CraftLab 기반 구조 및 라우팅

## 목표

- `PageName`에 `craft`가 등록되고 `renderPage()`에서 `CraftLab`이 렌더링된다
- `CraftSection` 컴포넌트가 존재하고, 기법명·설명·예시 영역·코드 스니펫 영역을 구조적으로 렌더링한다
- Labs 카테고리 헤더 내비게이션에 CraftLab 항목이 있고, 선택 시 CraftLab이 표시된다
- 빈 CraftLab 페이지(craft 0개)가 에러 없이 렌더링된다

## 구현 상세

### 접근 방법

기존 랩(StyleLab, MotionLab 등)의 파일 구조를 그대로 따른다. 단, CraftLab 내부는 DS 토큰 비교 UI가 아닌 섹션 스크롤 구조 — `LabLayout` + `LabSection`(Overview) + `CraftSection`(기법별)을 사용하되 토큰 조작 컨트롤은 없다.

### 변경 범위

**생성:**
- `src/app/pages/labs/CraftLab/CraftLab.tsx` — 메인 페이지 (빈 상태로 시작)
- `src/app/pages/labs/CraftLab/CraftLab.module.css`
- `src/app/pages/labs/CraftLab/CraftSection.tsx` — 개별 Craft 섹션 래퍼 (기법명, 설명, 예시 슬롯, 코드 스니펫 슬롯)
- `src/app/pages/labs/CraftLab/CraftSection.module.css`
- `src/app/pages/labs/CraftLab/index.ts`
- `src/app/pages/labs/CraftLab/crafts/` — 이후 craft들이 위치

**수정:**
- `src/app/pages/labs/index.ts` — `CraftLab` export 추가
- `src/app/config/nav-categories.ts` — labs 카테고리에 `craft` 항목 추가
- `src/App.tsx` — `PageName` 타입 + `renderPage()` switch case 추가
