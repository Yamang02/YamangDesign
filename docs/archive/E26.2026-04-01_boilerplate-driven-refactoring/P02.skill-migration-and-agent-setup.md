# P02: Skill 이식 및 Agent 체계 전환

## 목표
- vibe_boilerplate의 공통 skill 20개가 `.cursor/skills/`에 이식되어 있고, 각각 출처(`source: vibe_boilerplate@E26`)가 표기되어 있다
- `yamang-design-stack.md` skill이 프로젝트 특화 규칙(토큰 정책, CSS variable, 컴포넌트 구조)을 담고 있다
- `AGENTS.md`가 skill 라우팅 역할을 하며, 기존 `AGENT.md`와 `ai/rules.yaml`의 역할이 정리되어 있다
- boilerplate에 `templates/project-stack.tmpl.md` 템플릿이 존재한다

## 구현 상세

### 접근
1. vibe_boilerplate에 `templates/project-stack.tmpl.md` 작성 (프로젝트 특화 skill 뼈대)
2. YamangDesign `.cursor/skills/` 생성, 공통 skill 이식 (20개)
   - **워크플로우:** basis-skill-gate, epic-lifecycle, backlog-triage-and-epic-promotion, creative-design-brainstorm, implementation-plan-authoring, plan-execution-checkpoints, evidence-before-completion, branch-completion-options
   - **코드 품질:** layered-tdd-workflow, systematic-debugging, refactoring-workflow, local-sonarqube-agent-workflow, request-code-review, receive-code-review-rigor
   - **보안:** secure-coding-checklist, security-review
   - **스택:** typescript-stack, react-stack
   - **병렬 작업:** parallel-agent-dispatch, in-session-parallel-implementation, git-worktree-isolation
3. `yamang-design-stack.md` 작성 — `ai/rules.yaml` + `AGENT.md`에서 프로젝트 특화 규칙 추출
   - CSS variable 정책 (`var(--ds-*)`, `var(--ui-*)`)
   - 토큰 네이밍 (`--ds-{category}-{sub?}-{name}`)
   - 하드코딩 금지 정책 + 예외 조건
   - 컴포넌트 디렉토리 패턴 (`ComponentName/`)
   - path alias (`@domain`, `@app`, `@shared`)
4. `AGENT.md` → `AGENTS.md` 전환 (skill 라우팅)
5. 기존 `yamang-art-*` skill 유지
6. `ai/rules.yaml` 정리 — skill로 흡수된 부분 제거, ignore 파일 생성 로직만 유지

### 변경 범위
- **생성 (boilerplate):** `templates/project-stack.tmpl.md`
- **생성 (YamangDesign):** `.cursor/skills/` (공통 20개 + yamang-design-stack)
- **생성:** `AGENTS.md`
- **수정:** `ai/rules.yaml` (흡수된 규칙 제거)
- **삭제:** `AGENT.md` (AGENTS.md로 대체)
- **유지:** 기존 `yamang-art-*` skill
