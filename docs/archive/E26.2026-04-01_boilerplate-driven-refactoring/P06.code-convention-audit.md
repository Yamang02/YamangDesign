## 점검 결과 (2026-04-01)

### CN-01 네이밍

- **도메인 용어:** `palette`, `theme`, `token`, `semantic` 등 `src/domain/`에서 일관 사용.
- **레이어별 동사:** domain 유틸(`resolve`, `merge`, `create` 등), app 훅(`use*`) 패턴 **대체로 준수**.
- **boolean 접두사:** 주요 props/변수에 `is`/`has`/`can` 사용 — 전수 자동 판정은 미실시, 샘플링상 **큰 이탈 없음**.
- **타입 서픽스:** 앱 설정·저장 구조에 `StoredSettings` 등 의미 기반 이름 사용; DTO 강제 Request/Response 패턴은 **프론트 POC 수준**으로 전부 적용되지는 않음.

### CC-01 로직

- **`src/domain/` 내 `try {`:** grep 결과 **0건** — CC-03-02와 정합.
- **순수 함수:** 팔레트/매핑 유틸은 테스트와 함께 존재; ThemeProvider 등은 React·앱 의존( P04와 연계).

### CC-02 메서드·오케스트레이션

- **큰 훅** (`useGlobalSettings`, `useCustomThemePresets` 등): 저장소·상태 오케스트레이션 — 계산과 혼재 구간 **샘플링상 존재**, 완전 분리는 **미달** 가능.
- **Side effect:** `localStorage`는 `app/hooks`, `app/state`, `app/infra/storage` — domain 순수성 관점에서는 **ThemeProvider 주석**만 domain에 localStorage 언급.

### CC-03 에러

- **Domain try/catch:** 없음.
- **비즈니스 vs 시스템 에러 타입 분리:** 명시적 Result/Either 패턴 **미도입** (POC).

### CC-04 데이터 흐름

- **Domain이 `@app` 타입 참조:** P04와 동일 — **역방향 의존**으로 CC-04·AR와 연계 이슈.

### CC-05 코드 위생 (인라인 주석)

- **`//` 주석:** 다수 파일에 **설명용 한 줄 주석** 존재 (ThemeProvider, recommendations, Icon, labs 등). basis **CC-05 인라인 주석 금지**를 엄격 적용하면 **위반 목록 다수**로 분류 가능.
